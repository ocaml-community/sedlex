Module Sedlex_compiler.Ir

Intermediate representation for sedlex patterns.

This IR captures the regexp structure including named capture annotations (as bindings) but before tag allocation, fixed-length optimization, or discriminator handling. The compiler processes this IR to produce a compiled DFA with binding extraction information.

The IR does not depend on ppxlib and can be constructed directly in tests.

type t =
  1. | Chars of Cset.t
    (*

    Match a single code point in the given set.

    *)
  2. | Seq of t list
    (*

    Sequence (concatenation). Invariant: length >= 2. Use seq smart constructor.

    *)
  3. | Alt of t list
    (*

    Alternation. Invariant: length >= 2. Use alt smart constructor which flattens nested Alts.

    *)
  4. | Star of t
    (*

    Kleene star (zero or more).

    *)
  5. | Plus of t
    (*

    One or more.

    *)
  6. | Rep of t * int * int
    (*

    Rep (r, n, m): between n and m repetitions.

    *)
  7. | Eps
    (*

    Empty string (epsilon).

    *)
  8. | Capture of string * t
    (*

    Named capture: Capture (name, inner) wraps inner with an as binding. The compiler decides tag allocation strategy.

    *)

A pattern with named captures.

Smart constructors

Constructors that enforce structural invariants return result.

  • alt checks name consistency across branches.
  • capture checks for shadowed inner bindings.
  • seq rejects duplicate capture names across elements.
  • star, plus, rep reject inner captures.

val fixed_length : t -> int option

fixed_length t returns Some n if t always matches exactly n code points, or None if the length is variable.

All structural constraints are enforced by the smart constructors. check_invariant asserts these hold. Use for debugging.
