Regexp combinators

Regular expressions are built from combinators and compiled to a DFA.

type regexp

Abstract type of regular expressions.

val chars : Cset.t -> regexp

chars cset matches a single code point in cset.

val seq : regexp -> regexp -> regexp

seq r1 r2 matches r1 followed by r2 (concatenation).

val alt : regexp -> regexp -> regexp

alt r1 r2 matches r1 or r2 (alternation). When both operands are simple chars regexps, their character sets are merged into one.

val rep : regexp -> regexp

rep r matches zero or more repetitions of r (Kleene star).

val plus : regexp -> regexp

plus r matches one or more repetitions of r.

val repeat : regexp -> int -> int -> regexp

repeat r n m matches between n and m repetitions of r (bounded repetition). Requires 0 <= n <= m.

val eps : regexp

The empty regexp — matches the empty string (epsilon).

val compl : regexp -> regexp option

If the argument is a single chars regexp, returns a regexp which matches the complement set. Otherwise returns None.

val subtract : regexp -> regexp -> regexp option

If each argument is a single chars regexp, returns a regexp which matches the set (arg1 - arg2). Otherwise returns None.

val intersection : regexp -> regexp -> regexp option

If each argument is a single chars regexp, returns a regexp which matches the intersection set. Otherwise returns None.

Tagged DFA for as bindings

Named sub-match bindings (e.g. Star any as x) are implemented using tagged transitions in the DFA. Each as binding introduces up to two tags that record the start and end positions of the sub-match in the lexbuf's memory cells at runtime. When one boundary can be computed from a known offset, only one tag is needed (bind_start_only / bind_end_only); when both boundaries are known, no tags are needed at all.

Or-patterns (p1 as x) | (p2 as x) additionally use discriminator cells: integer values that record which branch was taken, so the PPX can extract the correct positions at match time.

type tag_op =
  1. | Set_position of int
    (*

    Set_position i: record the current lexbuf position in memory cell i.

    *)
  2. | Set_value of int * int
    (*

    Set_value (cell, v): record integer v in memory cell cell (used for or-pattern discriminators).

    *)

Tag operations emitted on DFA transitions.

val bind : regexp -> regexp * int * int

bind r wraps r with start/end tag epsilon nodes. Returns (wrapped_regexp, start_tag, end_tag) where start_tag and end_tag are the allocated memory cell indices.

val new_disc_cell : unit -> int

Allocate a fresh memory cell for an or-pattern discriminator.

val bind_disc : regexp -> int -> int -> regexp

bind_disc r cell value appends an epsilon node that sets cell to value after r matches. Used to tag each branch of an or-pattern so the PPX can tell which branch matched.

val bind_start_only : regexp -> regexp * int

Like bind, but only wraps with a start tag (no end tag). Used when the end position can be computed from a known offset.

val bind_end_only : regexp -> regexp * int

Like bind, but only wraps with an end tag (no start tag). Used when the start position can be computed from a known offset.

val reset_tags : unit -> unit

Reset the tag counter. Called before compiling each match%sedlex block.

DFA compilation

type dfa_state = {
  1. trans : (Cset.t * int * tag_op list) array;
    (*

    Each transition: (character set, target state, tag operations to execute when this transition fires).

    *)
  2. finals : bool array;
    (*

    finals.(i) is true if this state is accepting for rule i.

    *)
}
type dfa = dfa_state array

DFA states, indexed by state number. State 0 is the initial state.

type compiled = {
  1. dfa : dfa;
  2. init_tags : tag_op list;
    (*

    Tag operations to execute before entering the DFA (from epsilon closure of the initial NFA nodes).

    *)
  3. num_tags : int;
    (*

    Total number of memory cells needed at runtime. When num_tags = 0, no memory is allocated (pattern has no as bindings).

    *)
}

Result of compile.

val compile : regexp array -> compiled

compile rules determinizes the NFA for an array of regexp rules using subset construction. Returns the DFA, initial tag operations, and the total number of memory cells needed for as bindings. State 0 is always the initial state.

High-level compilation from IR

compile_ir takes an array of Ir.t patterns, handles tag allocation (including Start_plus/End_minus optimizations), discriminator insertion for or-patterns, and DFA construction. Returns the DFA and all information needed to generate binding extraction code.

type pos_expr =
  1. | Tag of {
    1. tag : int;
    2. offset : int;
    }
    (*

    Read memory cell tag and add offset.

    *)
  2. | Start_plus of int
    (*

    Position is n code points from the token start.

    *)
  3. | End_minus of int
    (*

    Position is n code points before the token end.

    *)

How to compute a sub-match boundary position at runtime.

type compiled_binding = {
  1. name : string;
  2. start_pos : pos_expr;
  3. end_pos : pos_expr;
  4. disc : (int * int) list;
    (*

    Discriminator conditions: (cell, value) pairs. Empty for simple (non-or) bindings. For or-patterns, branches with different positions have distinct values; branches with identical positions share a value.

    *)
}

Information about a single binding in the compiled output.

type compiled_ir = {
  1. dfa : dfa;
  2. init_tags : tag_op list;
  3. num_tags : int;
  4. bindings : compiled_binding list array;
    (*

    bindings.(i) is the list of binding info for rule i.

    *)
}

Result of compile_ir.

val compile_ir : Ir.t array -> compiled_ir

compile_ir rules compiles an array of IR patterns into a tagged DFA. Raises Assert_failure if invariant checking fails.

val dfa_to_dot : dfa -> string

dfa_to_dot dfa returns a Graphviz DOT representation of the DFA, including state labels, accepting state markers, transition character sets, and tag operations on edges.
